\section{Problem \& Motivation}

% SLIDE 3: THE PROBLEM
\begin{frame}{The Problem: Static Benchmarks $\neq$ Clinical Reasoning}
\begin{columns}[T]
\begin{column}{0.48\textwidth}
\begin{block}{Static QA Benchmarks}
\begin{itemize}
  \item MedQA, USMLE, PubMedQA
  \item \textbf{All} patient info given upfront
  \item Single-turn, one-shot answer
  \item GPT-4 exceeds \textbf{90\%} on USMLE
\end{itemize}
\end{block}
\end{column}
\begin{column}{0.48\textwidth}
\begin{block}{Real Clinical Reasoning}
\begin{itemize}
  \item \textbf{Interactive} \& \textbf{sequential}
  \item Iterative hypothesis formation
  \item Selective evidence acquisition
  \item Reasoning under \textit{uncertainty}
\end{itemize}
\end{block}
\end{column}
\end{columns}
\vspace{0.5em}
\begin{alertblock}{Gap}
High QA accuracy $\not\Rightarrow$ competence in actual multi-turn diagnostic consultations.
\end{alertblock}

\note{
Static benchmarks like MedQA give the model everything—history, vitals, labs—in a single prompt. The model just selects an answer. But real clinical reasoning is fundamentally different: physicians iteratively ask questions, form hypotheses, order tests, and refine their differential. A model scoring 90\% on USMLE tells us nothing about whether it can actually conduct a patient interview. This gap is what my entire thesis is about.
}
\end{frame}

% SLIDE 4: WHY THIS MATTERS
\begin{frame}{Why This Matters: Patient Safety}
\begin{itemize}
  \item \textbf{Existing systems are closed or flawed:}
  \begin{itemize}
    \item AMIE (Google): Closed-source, proprietary, non-reproducible
    \item Agent Hospital: Homogeneous agents $\rightarrow$ \alert{Lexical Leakage}
    \item MedAgents: Static, no multi-turn simulation
  \end{itemize}
  \vspace{0.5em}
  \item \textbf{Dangerous failure modes go undetected:}
  \begin{itemize}
    \item A model can appear \textit{confidently consistent} while being \textit{systematically wrong}
    \item Binary accuracy masks unsafe reasoning trajectories
    \item Cognitive biases amplify real-world clinical errors
  \end{itemize}
\end{itemize}

\vspace{0.5em}
\begin{block}{Core Thesis Argument}
Clinical AI must be evaluated on its \textbf{process}—not just its terminal outcome.
\end{block}

\note{
The stakes here are real—we're talking about systems that could inform clinical decisions. Google's AMIE showed promise but is closed-source and can't be reproduced or extended. Agent Hospital uses the same model for all agents, which I found introduces a fundamental vulnerability I call Lexical Leakage. And critically, evaluating only whether the final diagnosis is right or wrong doesn't tell you if the model ordered an MRI before checking blood pressure—that's an unsafe reasoning trajectory that accuracy alone can't capture.
}
\end{frame}

% SLIDE 5: RESEARCH QUESTIONS
\begin{frame}{Research Questions}
\begin{enumerate}
  \item[\textbf{RQ1.}] Can open-source LLMs (7B--8B) navigate multi-turn clinical consultations using local GPU inference rather than proprietary APIs?
  \vspace{0.6em}
  \item[\textbf{RQ2.}] Does \textit{Lexical Leakage} from homogeneous agent architectures artificially inflate diagnostic accuracy?
  \vspace{0.6em}
  \item[\textbf{RQ3.}] Can structured metacognitive reflection measurably improve diagnostic reasoning trajectories?
  \vspace{0.6em}
  \item[\textbf{RQ4.}] How vulnerable are open-source clinical LLMs to systematic cognitive bias injections (recency, confirmation)?
\end{enumerate}

\note{
These four questions drove every architectural and experimental decision. RQ1 establishes feasibility—can we do this without GPT-4 API costs? RQ2 addresses a vulnerability I discovered in existing frameworks. RQ3 tests my main architectural innovation—the reflection node. And RQ4 probes safety: if we deploy these models, how easily can their reasoning be corrupted by the same cognitive biases that affect human doctors?
}
\end{frame}

\section{Background Theory}

% SLIDE 6: POMDP
\begin{frame}{Theoretical Foundation: POMDP Formulation}
Clinical reasoning $\rightarrow$ Partially Observable Markov Decision Process

\vspace{0.3em}
\begin{block}{$M = \langle S, A, T, R, \Omega, O \rangle$}
\footnotesize
\begin{tabular}{@{}ll@{}}
  $S$ (State Space) & Hidden disease + physiological parameters \\
  $A$ (Action Space) & $\{a_{\text{dialogue}},\; a_{\text{test}},\; a_{\text{diagnosis}}\}$ \\
  $T$ (Transition) & Conversation progression over time $t$ \\
  $R$ (Reward) & Retrospective: DS, TR, IE metrics \\
  $\Omega$ (Observation) & Text from Patient \& Measurement agents \\
  $O$ (Obs.\ Function) & True state $\xrightarrow{\text{Patient persona}}$ NL observations \\
\end{tabular}
\end{block}

\vspace{0.3em}
\begin{center}
\begin{tikzpicture}[scale=0.85,
    state/.style={rectangle, rounded corners, draw=iitkgpblue, fill=lightbg, thick, minimum width=2cm, minimum height=0.8cm, font=\small},
    arr/.style={-{Stealth[length=5pt]}, thick, iitkgpblue}]
  \node[state] (s) at (0,0) {$s_t$ (Hidden)};
  \node[state] (a) at (4,0) {$a_t$ (Doctor)};
  \node[state] (o) at (8,0) {$\omega_t$ (Observed)};
  \draw[arr] (s) -- node[above,font=\scriptsize]{$O(s_t, a_t)$} (a);
  \draw[arr] (a) -- node[above,font=\scriptsize]{Patient/Meas.} (o);
  \draw[arr, bend left=30] (o) to node[below,font=\scriptsize]{$T(\cdot)$} (s);
\end{tikzpicture}
\end{center}

\note{
I formalised the entire clinical consultation as a POMDP. The true patient state—the actual disease—is hidden from the Doctor agent. The Doctor can only observe noisy text outputs from the Patient and Measurement agents. At each turn, it takes an action: ask a question, order a test, or render a diagnosis. The reward isn't optimised during the dialogue—it's computed retrospectively using my three custom metrics. This formulation is what makes the system mathematically rigorous rather than just a chatbot.
}
\end{frame}

% SLIDE 7: QUANTIZATION & LORA
\begin{frame}{Key Techniques: NF4 Quantization \& LoRA}
\begin{columns}[T]
\begin{column}{0.48\textwidth}
\begin{block}{NF4 Quantization}
\begin{itemize}
  \item Weights $\sim \mathcal{N}(0, \sigma^2)$ \\ $\rightarrow$ optimal 4-bit projection
  \item \textbf{Double quantization:} quantize the quantization constants
  \item 7B model: 14\,GB $\rightarrow$ \textbf{$\sim$4\,GB}
  \item Near-zero reasoning quality loss
\end{itemize}
\end{block}
\end{column}
\begin{column}{0.48\textwidth}
\begin{block}{LoRA (Low-Rank Adaptation)}
\begin{itemize}
  \item Freeze base $W_0$, train $\Delta W = BA$
  \item $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$; $r \ll d$
  \item My config: $r{=}16$, $\alpha{=}32$
  \item Targets: q, k, v, o projections
  \item Both adapters: $\sim$50\,MB total
\end{itemize}
\end{block}
\end{column}
\end{columns}
\vspace{0.4em}
\begin{center}
\small\textbf{QLoRA} = 4-bit NF4 base + bf16 LoRA adapters $\rightarrow$ train on consumer GPUs
\end{center}

\note{
These are the two key enabling technologies. NF4 quantization works because neural network weights naturally follow a normal distribution—so NormalFloat4 is the information-theoretically optimal 4-bit representation. Double quantization goes further by quantizing the quantization constants themselves, saving an additional 0.37 bits per parameter. LoRA lets me train only a small low-rank decomposition on top of the frozen base weights. The key insight is that I'm not just using these as black boxes—I engineered a system where the base model loads in 4-bit for inference, and the LoRA adapters are dynamically swapped depending on which LangGraph node is executing.
}
\end{frame}

\note{
\textbf{DEEP DIVE — Phase 1 Architecture (Fig 4.1):}

The flowchart on the left shows the full information flow. The Scenario Loader reads from agentclinic\_medqa.jsonl and partitions the data: the Doctor only gets the consultation objective (e.g., ''Evaluate patient with chest pain''), the Patient gets the full symptom profile, the Measurement Agent gets the lab results, and the Moderator gets the ground-truth diagnosis. This strict partitioning is the OSCE (Objective Structured Clinical Examination) structure used in real medical education.

\textbf{The while-loop problem:} The orchestrator was a Python while loop: while not diagnosis\_ready: route\_message(). This is inherently fragile. When an open-source model produces a malformed stop token, or forgets to include DIAGNOSIS READY in its output, the loop either crashes or runs forever. There is no state recovery. In experiments, I observed silent skips where a scenario would terminate without a valid diagnosis — contaminating the accuracy measurement.

\textbf{Why these four models?} Coverage across different training paradigms:
- Mistral-7B: General-purpose instruction tuning (RLHF-style) on diverse web corpora
- BioGPT: Masked language modeling pretraining on 30M PubMed abstracts — focuses exclusively on biomedical text
- JSL-Med-Llama-3: Supervised fine-tuning on clinical NLP datasets (clinical notes, MIMIC, medical QA)
- Apollo: General text, no medical specialization — serves as negative control

\textbf{Catastrophic forgetting — the mechanism:} When you fine-tune a pretrained LLM on a narrow domain dataset (e.g., medical literature QA), you update the weights to maximize the likelihood of the training examples. But the training signal for the general-purpose conversational patterns (turn-taking, follow-up reasoning, instruction following) is absent in medical QA data. The weights for those circuits get overwritten. Result: the model becomes better at answering ''What is the treatment for atrial fibrillation?'' but worse at ''Given what the patient just said, what should I ask next?'' The multi-turn diagnostic task requires BOTH — and general Mistral-7B has the latter intact.

\textbf{Expected examiner Q\&A:}
Q: ''Why not use GPT-4 as the Doctor?''
A: ''The thesis specifically studies open-source LLMs that can run on local hardware without API costs. GPT-4 is a black box — we can't modify its weights, inject biases in a controlled way, or train LoRA adapters on it. Local models are essential for the experimental control we need.''

Q: ''BioGPT is only 1.5B parameters — isn't the comparison unfair?''
A: ''Yes, parameter count is a confound. But even controlling for this, the pattern holds: domain fine-tuning tends to hurt multi-turn reasoning. JSL-Med-Llama-3 is 8B — comparable to Mistral-7B — and still performs 12 percentage points worse. The catastrophic forgetting effect is real regardless of model size.''
}
\end{frame}

\note{
\textbf{Reading the Phase 1 Bar Chart:}

This chart plots diagnostic accuracy for the four models across 107 MedQA scenarios. The y-axis represents the percentage of scenarios where the Moderator Agent judged the Doctor's final diagnosis as semantically equivalent to the ground truth.

\textbf{Why 107 scenarios?} The full AgentClinic-MedQA dataset has more cases, but running 107 scenarios × 4 models × 10 turns each already represents thousands of LLM inference calls. At ~30 seconds per inference step on the H100, a single experiment takes several hours. 107 was calibrated for statistical significance while remaining computationally feasible.

\textbf{Bias results from Phase 1 (on best model Mistral-7B):}
- No bias: 43.9\%
- Recency bias: 33.6\% (drop of 10.3 percentage points)
- Confirmation bias: 31.8\% (drop of 12.1 percentage points)
These preliminary Phase 1 bias results motivated the more rigorous E4/E5 experiments in Phase 2 using the domain-specialist JSL-MedMistral model.

\textbf{The accuracy metric itself:}
Accuracy = (Number of Correct Diagnoses) / 107 × 100
A diagnosis is ''correct'' if the Moderator Agent (running the same LLM as a semantic judge) outputs ''Yes'' when comparing the Doctor's diagnosis to the ground truth. This uses LLM-mediated semantic equivalence rather than exact string matching — so ''Heart Attack'' = ''Myocardial Infarction'' = ''AMI'' = correct.

\textbf{Potential examiner critique:}
Q: ''The Moderator uses the same type of LLM as the Doctor. Doesn't that bias the evaluation?''
A: ''This is a acknowledged limitation. The Moderator is instructed specifically to judge semantic equivalence with a yes/no output, which is a much simpler task than clinical diagnosis. In practice, the Moderator rarely makes errors on unambiguous cases. For ambiguous cases, this is equivalent to how clinical examiners in real OSCEs exercise judgment about synonymous terminology. In future work, a rule-based ontology cross-referencing ICD-10 codes would eliminate this concern.''
}

% SLIDE 9: LEXICAL LEAKAGE
\begin{frame}{Original Discovery: Lexical Leakage}
\begin{center}
\begin{tikzpicture}[scale=0.8,
    agent/.style={rectangle, rounded corners, draw=iitkgpblue, fill=lightbg, thick, minimum width=3cm, minimum height=1.2cm, font=\small\bfseries, align=center},
    model/.style={ellipse, draw=alertred, fill=alertred!10, thick, minimum width=2.8cm, font=\small\bfseries},
    arr/.style={-{Stealth[length=5pt]}, thick}]

  \node[agent] (doc) at (-3, 1.5) {Doctor\\Agent};
  \node[agent] (pat) at (3, 1.5) {Patient\\Agent};
  \node[model] (wts) at (0, -0.8) {Same Weights $W$};

  \draw[arr, alertred, line width=1.5pt] (wts) -- (doc) node[midway, left, font=\scriptsize, text=alertred]{shared embeddings};
  \draw[arr, alertred, line width=1.5pt] (wts) -- (pat) node[midway, right, font=\scriptsize, text=alertred]{shared embeddings};
  \draw[arr, dashed, alertred, line width=1.5pt] (pat) -- (doc) node[midway, above, font=\scriptsize, text=alertred]{\textbf{Implicit information leakage}};
\end{tikzpicture}
\end{center}
\vspace{-0.3em}

\begin{block}{Definition (My Contribution)}
When both Doctor and Patient share the same model weights, they project text into \textbf{identical latent embedding manifolds}. The Doctor can ``recognise'' the mathematical structure of the Patient's generated text, gaining \textbf{implicit access to ground-truth information}.
\end{block}

\vspace{0.3em}
$\rightarrow$ \textbf{Solution:} Heterogeneous multi-engine architecture (separate model instances)

\note{
This is a vulnerability I identified and named. When you use the same 7B model for both the Doctor and Patient, both agents encode text into the same embedding space. The Doctor doesn't just read the Patient's words—it recognises the statistical fingerprint of how that model generates medical text. This means the 43.9\% accuracy in Phase 1 was partially inflated by this implicit information sharing. I proved this by showing that accuracy actually increased to 45.8\% after switching to separate models—the metacognitive reflection node more than compensated for removing this artificial advantage.
}
\end{frame}

\section{Enhanced Framework Architecture}
% ============================================================

% SLIDE 11: LANGGRAPH DCG
\begin{frame}{LangGraph Directed Cyclic Graph State Machine (Fig.\ 5.1)}
\begin{columns}[T]
\begin{column}{0.54\textwidth}
  \includegraphics[width=\linewidth]{LangGraph}
  \centering\tiny\textit{Fig 5.1: DCG topology — reflection node executes before every spoken turn}
\end{column}
\begin{column}{0.44\textwidth}
\footnotesize
\begin{block}{Graph Topology}
\texttt{Entry} $\to$ \texttt{doctor\_reflection} $\to$ \texttt{doctor\_speaker}\\[2pt]
\texttt{doctor\_speaker} $\xrightarrow{\text{dialogue}}$ \texttt{patient}\\
\texttt{doctor\_speaker} $\xrightarrow{\text{REQUEST TEST}}$ \texttt{measurement}\\
\texttt{doctor\_speaker} $\xrightarrow{\text{DIAGNOSIS READY}}$ \texttt{END}\\[2pt]
\texttt{patient} $\to$ \texttt{doctor\_reflection} (loop)\\
\texttt{measurement} $\to$ \texttt{doctor\_reflection} (loop)
\end{block}
\vspace{0.3em}
\begin{itemize}
  \item[\checkmark] Deterministic routing — no ambiguous transitions
  \item[\checkmark] Reflection executes \textbf{before every turn}
  \item[\checkmark] \texttt{ClinicalState} TypedDict: differential trajectory, tests ordered, full dialogue log
  \item[\checkmark] Non-destructive \textbf{reducers} accumulate evidence
\end{itemize}
\end{column}
\end{columns}
\note{
\textbf{DEEP DIVE — LangGraph DCG Architecture (Fig 5.1):}

The image shows the actual LangGraph topology from the implementation. LangGraph is a graph execution framework built on top of LangChain that allows defining nodes (Python functions) and edges (routing logic) as a directed graph. Unlike a simple while-loop, the state machine guarantees that every state transition is explicit, deterministic, and auditable.

\textbf{What is a TypedDict and why does it matter?}
ClinicalState is defined as a Python TypedDict — a type-annotated dictionary. This gives us compile-time type checking and ensures every node knows exactly what keys exist in the state. The fields are:
- last\_input\_to\_doctor: str — the most recent patient or measurement response
- last\_doctor\_message: str — what the doctor just said
- differential\_diagnoses: List[str] — current top-3 DDx from reflection
- turn\_count: int — enforces N\_max\_infs = 10 termination
- diagnosis\_ready: bool — terminal flag
- differential\_trajectory: List[List[str]] — accumulated DDx snapshots for DS computation
- tests\_ordered: List[str] — accumulated test names for TR computation
- full\_dialogue: List[dict] — complete conversation with timestamps for analysis

\textbf{Reducers and non-destructive updates:} LangGraph reducers define HOW node outputs are merged into the state. I used the list concatenation operator (operator.add) as the reducer for trajectory fields — this means each node APPENDS to the existing list rather than overwriting it. This is critical for maintaining the full trajectory history without losing intermediate states.

\textbf{The routing function (route\_doctor):} This is a Python function that receives the current ClinicalState and returns a string indicating which node to execute next:
- If diagnosis\_ready or turn\_count >= 10: return END
- If ''REQUEST TEST'' in last\_doctor\_message.upper(): return ''measurement''
- Otherwise: return ''patient''
This routing is pure Python logic — it's deterministic given the same state, unlike the while-loop's ad-hoc conditionals.

\textbf{Why LangGraph specifically?}
Alternatives: LangChain ConversationChain (no graph structure), AutoGen (opinionated multi-agent, harder to customize), raw Python loops (non-deterministic). LangGraph was chosen because: (1) it enforces explicit state schemas, (2) it supports cyclic graphs natively, (3) it integrates with HuggingFace model wrappers cleanly, (4) it provides execution graphs you can visualize and debug.

\textbf{Expected Q\&A:}
Q: ''Why not just use AutoGen or CrewAI for multi-agent orchestration?''
A: ''AutoGen and CrewAI are opinionated frameworks with pre-built agent roles. For this thesis, I needed fine-grained control: custom bias injection into system prompts, trajectory capture at every node, dynamic LoRA adapter swapping between graph nodes, and custom routing logic. LangGraph is an execution substrate — it doesn't make assumptions about what the agents do, just how they're connected. That flexibility was necessary for the experimental design.''

Q: ''What happens if the graph exceeds the max turn count?''
A: ''The routing function catches this: if turn\_count >= total\_inferences, it returns END regardless of whether DIAGNOSIS READY was emitted. Then the evaluation code extracts whatever is in last\_doctor\_message as the final answer. There's also a forced diagnostic closure mechanism: if the doctor's last message doesn't contain DIAGNOSIS READY, a second aggressive prompt with few-shot examples forces the model to commit to a diagnosis.''
}
\end{frame}

% SLIDE 12: METACOGNITIVE REFLECTION
\begin{frame}{Key Innovation: Latent Metacognitive Reflection Node}
\begin{columns}[T]
\begin{column}{0.55\textwidth}
\textbf{Problem:} LLMs forced to simultaneously reason \emph{and} converse produce hallucinations and chain-of-thought leaks.

\vspace{0.5em}
\textbf{Solution:} Decouple reasoning from generation.

\vspace{0.3em}
\begin{enumerate}
  \item \texttt{doctor\_reflection} generates JSON differential at $T{=}0.1$
  \begin{itemize}
    \item[\texttt{>}] \texttt{["Asthma", "COPD", "Pneumonia"]}
  \end{itemize}
  \item Stored in \texttt{ClinicalState} trajectory log
  \item \textbf{NOT} injected into patient-facing dialogue

\note{
\textbf{DEEP DIVE — Multi-Engine Architecture \& Quantization:}

To eliminate Lexical Leakage, I needed separate model instances for each role. But loading three 7B models in 16-bit float (FP16) requires 3 × 14 GB = 42 GB. While this fits on the 96GB H100, memory also needs to be allocated for the KV-cache (which grows linearly with context length) and attention matrices. More importantly, making it run in less VRAM makes the framework accessible to consumer GPUs (like RTX 3090/4090 with 24GB).

\textbf{How NF4 Quantization works:} Weights in trained neural networks normally follow a Gaussian (normal) distribution centered around zero. Standard integer quantization (INT4) divides the min-max range evenly, meaning most values cluster in the middle bins, wasting the outer bins. NormalFloat4 (NF4) uses a non-linear spacing derived from the quantiles of a standard normal distribution. This means each bin has an equal probability of containing a value, maximizing information-theoretic retention. The result is a 71\% memory reduction (14GB to ~4GB) with virtually zero degradation in perplexity or downstream accuracy. Double quantization goes further by quantizing the scaling factors themselves, saving an extra 0.37 bits/parameter.

\textbf{Cognitive Asymmetry:} The Doctor needs deep biomedical knowledge (JSL-MedMistral-7B), but the Patient just needs to act like a normal human reporting symptoms (generalist Mistral-7B). This asymmetry naturally prevents Lexical Leakage because the models project into different mathematical vector spaces. To save even more memory, the Moderator agent (which only runs exactly once at the end to evaluate semantic equivalence) shares the Patient engine's model weights, requiring 0 extra GB.

% SLIDE 15: PROCESS METRICS
\begin{frame}{Novel Process-Oriented Clinical Metrics}
\vspace{-0.3em}
\begin{block}{1. Diagnostic Stability (DS) — Jaccard Similarity}
\vspace{-0.5em}
\[\text{DS} = \frac{1}{T - 1} \sum_{t=1}^{T-1} \frac{|\text{DDx}_t \cap \text{DDx}_{t+1}|}{|\text{DDx}_t \cup \text{DDx}_{t+1}|}\]
\vspace{-0.5em}
\small High $\rightarrow$ stable refinement \quad|\quad Low $\rightarrow$ erratic hypothesis switching
\end{block}

\begin{block}{2. Test Rationality (TR) — Clinical Protocol Compliance}
\vspace{-0.5em}
\[\text{TR} = \begin{cases} 0 & \text{if } \exists\, t_{\text{adv}} \text{ ordered before any } t_{\text{basic}} \\ 1 & \text{otherwise} \end{cases}\]
\vspace{-0.5em}
\small Basic workup (CBC, vitals) must precede advanced tests (MRI, biopsy)
\end{block}

\begin{block}{3. Information Efficiency (IE) — Hypothesis Diversity}
\vspace{-0.5em}
\[\text{IE} = \frac{N_{\text{unique}}}{T}\]
\vspace{-0.5em}
\small Unique differential hypotheses generated per turn
\end{block}

\note{
\textbf{DEEP DIVE — Process-Oriented Metrics Formulation:}

Static benchmarks only measure if the final answer is right (accuracy). I defined three metrics to evaluate the \textit{trajectory} of reasoning.

\textbf{DS (Diagnostic Stability):} This measures how drastically the model's internal hypothesis changes from turn to turn. $DDx_t$ is the set of top-3 differential diagnoses at turn $t$. The formula is the average Jaccard Similarity coefficient between consecutive turns. A Jaccard similarity of 1 means the exact same set of 3 diseases was considered in turn t and t+1. A value of 0 means completely disjoint sets. 
\textit{Why it matters:} A human doctor starts broad and narrows down. If a model has a DS of 0.2, it is ''hypothesis thrashing'' — jumping from pneumonia to appendicitis to eczema wildly with every piece of new information.

\textbf{TR (Test Rationality):} Medical protocol demands cost-effective, least-invasive testing first. You don't order a brain MRI before taking a temperature. I categorized the MedQA tests into 'basic' (vitals, CBC blood test) and 'advanced' (CT scans, biopsies, etc). TR is a boolean flag: if the model orders an advanced test before ever ordering a basic one, it fails (0) for that scenario.

\textbf{IE (Information Efficiency):} Measures the total number of unique hypotheses generated across the entire conversation divided by the total turns $T$. High IE (>1.5) means the model is actively exploring a wide differential space. Low IE (<0.8) means the model is fixated on a few ideas from the very beginning.

\textbf{Q\&A:}
Q: ''Why Jaccard similarity for DS instead of something like semantic cosine similarity?''
A: ''Jaccard was chosen for its strict and easily interpretable bound [0, 1] on exact string matches (since the outputs are constrained JSON strings). However, as I note in the limitations, Jaccard penalizes synonyms. If the model outputs 'Heart Attack' then 'Myocardial Infarction', Jaccard scores 0, whereas BioBERT embeddings would show high cosine similarity. This is a known limitation of my metric and direct future work.''
}
\end{frame}

\note{
\textbf{DEEP DIVE — Bias Impact Mechanism:}

The charts show exactly how badly biases degrade accuracy and test rationality relative to the unbiased E2 baseline (dashed red line). But \textit{why} does this happen mathematically?

\textbf{Recency Bias Mechanics:} Transformer models use a self-attention mechanism ($Attention(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$). When $k=5$ recent case summaries are injected into the context window, the attention scores for those recent tokens artificially inflate because they match the general semantic distribution of 'patient case'. The LLM conflates the historical context with the active context. The model starts diagnosing the current patient with the disease of the previous patient.

\textbf{Confirmation Bias Mechanics:} LLMs generate text autoregressively by sampling from a probability distribution over the vocabulary. Priming the system prompt with 'you are confident the patient has X' skews the initial latent representations. As the model generates text confirming this, the generated text is fed back into its own context window. Because LLMs suffer from 'sycophancy' (agreeing with their own context), this creates a runaway feedback loop. Alternative hypotheses are pushed down in the softmax probabilities, eventually rounding down to zero. The model literally cannot 'see' the contradictory evidence in the patient's test results because its attention is overly concentrated on its own confirmatory statements.

\textbf{Q\&A:}
Q: ''Do human doctors experience these exact same biases?''
A: ''Yes. The scenarios I designed mimic ecological reality. A doctor seeing five consecutive flu cases in an ER is highly likely to diagnose the sixth patient with the flu (recency). A doctor reading an ominous referral note may immediately anchor on cancer and ignore alternative benign causes (confirmation/anchoring). The point is that LLMs, without explicit safeguards, replicate these human frailties perfectly.''
}

- The Reasoning pairs take a patient's history and map it strictly to a JSON array of differential diagnoses.
- The Dialogue pairs take a patient's statement and map it to an empathetic, medically relevant follow-up question.

\textbf{Mathematical foundation of LoRA:} Normal fine-tuning updates the entire massive weight matrix $W$. LoRA freezes $W_0$ and adds a small parallel update matrix $\Delta W = BA$, where $B$ and $A$ are low-rank matrices. For a layer dimension of $d=4096$, configuring $r=16$ vastly reduces the parameters from 16 million to 131 thousand. I targeted all query, key, value, and output attention projections. The base model is quantized to 4-bit, but the adapters are trained and loaded in 16-bit brain float (bf16).

\textbf{Dynamic Swapping (The hack):} At inference time, the LangGraph state machine controls the adapters. Right before the doctor\_reflection node fires, I call `model.set\_adapter(''reasoning'')`. Right before the doctor\_speaker node fires, I call `model.set\_adapter(''dialogue'')`. For the patient node, I call `model.disable\_adapters()`. Because the adapters are so small (~25MB each), this pointer swap relies purely on VRAM bandwidth and adds practically zero latency to generation.

\textbf{The Result:} The chart on the left (`lora\_comparison.pdf`) shows the jump from E2 (JSL-MedMistral without adapters) to E6 (with adapters). E6 improves accuracy by +4.6 percentage points, matching or beating massive 70B models in other benchmarks. It works.
}
\end{frame}

% SLIDE 23: LIMITATIONS
\begin{frame}{Limitations \& Self-Critique}
\begin{enumerate}
  \item \textbf{DS uses textual Jaccard:} Penalises valid medical synonyms\\
  {\small\color{gray} ``Myocardial Infarction'' $\neq$ ``Heart Attack'' $\rightarrow$ false instability}\\
  {\small\color{accentteal} Fix: BioBERT semantic cosine similarity}
  \vspace{0.3em}

  \item \textbf{TR assumes $\geq$1 test ordered:} Model that guesses without any tests gets TR=1\\
  {\small\color{accentteal} Fix: Condition TR on $N_{\text{tests}} \geq 1$}
  \vspace{0.3em}

  \item \textbf{IE is unbounded:} Hallucinated hypotheses can inflate score\\
  {\small\color{accentteal} Fix: Cap numerator to clinically plausible hypotheses}
  \vspace{0.3em}

  \item \textbf{Text-only Measurement Agent:} Real diagnostics need images (X-rays, ECGs)
  \vspace{0.3em}

  \item \textbf{Single dataset:} Only AgentClinic-MedQA (107 scenarios)\\
  {\small\color{accentteal} Needs: MIMIC-IV, NEJM case validation}
\end{enumerate}

\note{
\textbf{DEEP DIVE — Navigating Limitations:}

Every good defense shows intellectual honesty. 

\textbf{The DS Jaccard Issue:} If the model's differential is `['Lung Cancer']`, and next turn it is `['Pulmonary Neoplasm']`, Jaccard intersection is 0, so DS drops. Clinically, they are the same. A better approach would be embedding both sets of text using a specialized encoder like BioBERT, and calculating the cosine similarity between the vector centroids.

\textbf{The TR Edge Case:} Test Rationality flags when advanced tests are ordered before basic ones. What if the doctor asks three questions, orders zero tests, and confidently hallucinates a diagnosis? Technically, no advanced test preceded a basic test. TR evaluates to 1 (perfect). This is a logic flaw. The denominator of the workflow needs to assert that at least one basic test must occur.

\textbf{Measurement Modality:} In real life, a doctor doesn't just read "The chest X-ray shows bibasilar crackles opacities." They look at the actual film. The current framework is text-bound.

\textbf{Q\&A:}
Q: ''Are you worried the dataset was in the model's pretraining data?''
A: ''Yes, data contamination is pervasive in LLMs. However, the MedQA dataset in its raw form is a multiple-choice question. The AgentClinic framework refactors this radically into a dynamic prompt schema where information is hidden. Even if the model 'memorized' the MedQA question, the sequential hidden-state nature of our simulation forces it to perform novel reasoning to retrieve that state.''
}
\end{frame}

\note{
\textbf{DEEP DIVE — Future Directions:}

These future directions all build directly on the infrastructure I've already created. 

\textbf{Process Rewards (DPO):} DPO (Direct Preference Optimization) is a technique that replaces RLHF by directly framing reward modeling as a classification loss. Currently, LLMs are trained to output the correct final diagnosis. Using the trajectory data from the LangGraph, we can generate a pair of trajectories: $y\_win$ (a dialogue with high Test Rationality and moderate DS) and $y\_lose$ (a dialogue that achieves the exact same correct diagnosis, but ordered invasive tests unnecessarily). Training via DPO on these pairs will align the model to prefer clinically safe reasoning patterns, not just correct answers.

\textbf{RLAIF (Reinforcement Learning from AI Feedback):} The AgentClinic simulation acts as an open-ended gym environment. The ClinicalTrajectoryEvaluator provides a programmatic reward signal at the end of the episode. We can use Proximal Policy Optimization (PPO) to train the Doctor agent against the Patient agent over thousands of episodes without human-in-the-loop annotations.

\textbf{Debiasing Adapters:} If I can train an adapter for 'reasoning', I can train one for 'recency-debiasing'. When the framework detects erratic DS (hypothesis thrashing indicative of recency bias), the LangGraph router can dynamically load the debiasing LoRA adapter to actively counteract the cognitive distortion in real-time.
}
